Functional Chess Game

This project was motivated by our groups chess rules and funtionality fascination! This is for the fun of the game and does not aim to solve any issue but to test our OOP coding ability!

This project is a functional chess game, that a user can play, it uses a double array, so a matrix, to create a full chess board, users are, at the moment, able to play by knowing the board in their heads by entering the piece and the x,y coordinate posiiton on the chess board

The project will be made in Java and as said above will feature an x,y coordinate system as its main user interface, it will be terminal printed or potentially have a gui representation however that might be a reach goal, we are aiming to have the user type the piece and the position it moves to and everytime it moves, do a board check to see if either king is in check. We will also be looking at pathlines of the pieces to determine if they will be blocked by other pieces, captured, en passant, or if a pawn made it all the way to the end of the enemies board.

The implementation was made by starting off with test cases to determine funtionality and expected behavior, these tetss will be updated as we work on the code and will continue to create expected behavior. We created classes for each of the pieces with a large overarching Abstract piece class to determine general piece functionality, we also have a board class which deals with the general board architectire, creation of the matrix for which the pieces exist. I worked stubs and testing for the pieces as each piece has different implementation and thus must have different edge cases to make sure our implementation functions as normal chess is expected to function!
